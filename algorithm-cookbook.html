Master Theorem for Runtime on Recursive Algorithms

T(n) = a * T(n⁄b) + O(nd)

  1. Identify a, b, &ambp; d
  2. Apply table below to solve for runtime
Value of d T(n)
d > logb a O(nd)
d = logb a O(nd lg n) where lg = log2
d < logb a O(nlogb a)

Useful to remember:
O(n0) = O(1)
 logx x = 1
logx 1 = 0
 O(n + a)b = O(n)b

Insertion Sort

Input: unsorted array A[1...n] of size n
Output: sorted array A[1...n]
Structure: uses two indices (i and j) and two loops: for j=2 to n, while i>0 and A[i]<key
Maintains: key=A[j] for current value that is being inserted (i.e. sorted)
Run-time: O(n2)

Proof for Insertion Sort

Loop Invariant: at start of each for loop all elements in A[1...j-1] are sorted

Base Case: at start j=2 and A[1...j-1] = A[1] ∴ single element is sorted

Induction Hypothesis (IH): assume true for j when 2 ≤ j ≤ k

Inductive Step (IS): Consider when j = k+1. Item j = k+1 is only inserted at A[k+1] if A[j] ≥ A[k]. By IH A[1...k] is sorted, so this maintains sorted order. Otherwise if A[j] ≤ [k], then each element in A[1...k] is examined in reverse order until the ith element is found where A[k+1=j] > A[i]. In the resulting array, A[1...i] is sorted, A[i+1] = A[j], and A[i+2...j-1] is sorted. Therefore, sorted order is maintained.

Pseudocode for Insertion Sort



Insertion-Sort(A[1..n]) where A is unsorted array

01 for j=2 to n

02  key = A[j]

03  i = j-1

04  while i>0 and A[i]>key

05      A[i+1] = A[i]

06      i = i-1

07  A[i+1] = key

Merge Sort

Input: unsorted array A[1...n], left index p, right index r
Output: sorted array A[1...n]
Structure: recursion, split array in half until single itm, then merge back together in sorted order
Maintains: q = ⌊p+r⁄2⌋ for middle index of array
Run-time: O(n lg n) where lg = log2

Recursive Call: 2 (n⁄2)
Merge Call: O(n)
Total Runtime: T(n) = 2 (n⁄2) + O(n)
Reduce: apply Master Theorem where a=2, b=2, d=1 to arrive at O(n lg n) where lg = log2

Pseudocode for Merge Sort



Merge-Sort(A[1..n], p, r]) where A is unsorted array

01 if p < r

02  q = ⌊p+r⁄2⌋

03  MergeSort(A, p, q)

04  MergeSort(A, q+1, r)

05  Merge(A, p, q, r)

Merge

Input:
Either:
* Two sorted arrays A1 and A2 or
* Single array with markers: left index p, middle index q, right index r

Output: single sorted array (input arrays combined)
Structure:
1. creates copies of input arrays with sentinels added to each input array
2. loops for k=p to r and updates A[k] to min value of L & R

Maintains:
* n1 = q - p + 1 to track section of input array
* n2 = r - q to track remaining section of input array
* L[1…n1 + 1], R[1…n2 + 1] for copies of input array with extra data location for sentinels
* i=1, j=1 for markers for location in L & R during (k = p to r) loop

Run-time: O(n) // 3 for loops: n1, n2, p to r = n

Pseudocode for Merge



Input: Single array with markers to sub-divide

Merge(A, p, q, r]) where A is unsorted array

01 n1 = q - p + 1  // p to q inclusive of q

02 n2 = r - q  // q to r exclusive of q

03 L[1…n1+1], L[1…n2+1] are new arrays  // need extra memory allocation for sentinels

04 for i=1 to n1  // fill L

05  L[i] = A[p+i-1]

06 for j=1 to n2  // fill R

07  R[j] = A[q+j]

08 L[n1+1] = ∞  // add sentinel

09 R[n2+1] = ∞  // add sentinel

10 i=1, j=1 // reset i & j

11 for k=p to r  // loop and assign min value

12  if L[i] ≤ R[j]

13      A[k] = L[i]

14      i = i+1

15  else

16      A[k] = R[j]

17      j = j+1




Input: Two sorted arrays
Merge(A1[1...m], A2[1...n]) where A1 and A2 are sorted arrays
01 L[1...m+1], R[1...n+1], A[1...m+n] are new arrays  // need extra memory allocation for sentinels
02 for i=1 to m  // fill L
03  L[i] = A1[i]
04 for j=1 to n  // fill R
05  R[j] = A2[j]
06 L[m+1] = ∞  // add sentinel
07 R[n+1] = ∞  // add sentinel
08 i=1, j=1 // reset i & j
09 for k=1 to (m+n)
10  if L[i] ≤ R[j]
11      A[k] = L[i]
12      i = i+1
13  else
14      A[k] = R[j]
15      j = j+1
16 return A

Quicksort

Input: unsorted array A[1…n], left index p, right index r
Output: sorted array A[1…n]
Structure:
1. Partitions (semi-sorts) last item in array q
2. Recursively calls itself on lower and upper parts of array aroud q (sorted partition element)
3. If randomized, then partition call is to Randomized-Partition

Maintains: single element that is in sorted position, marked as either:
* q = Partition(A, p, r) or
* q = Randomized-Partition(A, p, r)

Run-time:
* Worst Case: O(n2) for both non-randomized and randomized
* Expected Case: O(n lg n) for randomized only where lg = log2

Pseudocode for Quicksort



Quicksort(A[1…n], p, r) where A is unsorted array

01 if p < r

02  q = Partition(A, p, r)  // T(n) = O(n)

03  Quicksort(A, p, q-1)  // T(n) ranges from O(n lg n) to O(n2)

04  Quicksort(A, q+1, r)  // T(n) ranges from O(n lg n) to O(n2)




Random-Quicksort(A[1...n], p, r) where A is unsorted array
01 if p < r
02  q = Random-Partition(A, p, r)  // T(n) = O(n)
03  Random-Quicksort(A, p, q-1)  // T(n) ranges from O(n lg n) to O(n2)
04  Random-Quicksort(A, q+1, r)  // T(n) ranges from O(n lg n) to O(n2)


Random-Partition(A, p, r) where A is unsorted array
01 x = Random(p,r)  // random index between p and r inclusive
02 exchange A[x] and A[r]
03 return Partition(A, p, r)

Partition

Input: unsorted array A[1…n], left index p, right index r
Output: the index of sorted pivot element with array semi-sorted around it
Structure:
1. Marks pivot element x = A[r]
2. Marks i = p-1 at outer left of array
3. Loops for j = p to r-1
* Swaps elements so that A[1…i] < x < A[i+1…j-1]

Loop Invariant: At the start of each for loop:
* all elements in A[p…1] ≤ x
* all elements in A[i+1…j-1] > x
* all elements in A[j...r-1] are unsorted

Run-time: O(n) // loops through each element once

Pseudocode for Partition



Partition(A, p, r) where A is unsorted array

01 x = A[r]  // set pivot

02 i = p-1  // set left marker i outside of p

03 for j=p to r-1  // loop from p to pivot

04  if A[j] ≤ A[i]

05      i = i+1

06      exchange A[i], A[j]

07 exchange A[i+1], A[r]  // swap pivot

08 return i+1  // return pivot index

Binary Search

Input: a sorted array, value x to find
Output: the index position of value x in array
Structure: checks middle element and recursively searches left or right as appropriate
Run-time: O(lg n) where lg = log2

IMPORTANT: Always SORT before using Binary Search!

Pseudocode for Binary Search



Binary-Search(A, x) where A is a SORTED array

01 return Recursive-Search(A, 1, n, x)  // initial call, assumes 1 is left-most index




Recursive-Search(A, p, r, x) where A is a SORTED array, p and r are left and right markers, and x is element to find within A
01 if p == r and A[p] == x  \\ single element in array
02  return p
03 if p < r
04  q = ⌊p+r⁄2⌋
05  if A[q] == x
06      return q  // found x
07  if x < A[q]
08      return Recursive-Search(A, p, q, x)  \\ x is located in L half
09  else 
10      return Recursive-Search(A, q+1, r, x)  \\ x is located in R half, careful to use q+1
11 return -1  // x not found

Select

Input: unsorted array A, order statistic i that seeks ith smallest element (1 ≤ i ≤
n)
Output: the value of the ith smallest element (i.e. the element x that is larger than exactly i-1 other elements)
Structure:
1. Partitions to semi-sort an element (random or median of medians)
2. Partition returns an index:
* if the index matches i then return that value
* else recursively call on upper or lower half as appropriate

Run-time:
* Worst Case for Randomized-Select: O(n2)
* Expected Case for Randomized-Select: O(n)
* Worse Case for Deterministic-Select: O(n)

Pseudocode for Select



Random-Select(A, p, r, i) where A is unsorted array

01 if p == r

02  return A[p]

03 q = Random-Partition(A, p, r)

04 k = q - p + 1  // important: # of elements in (q-p) inclusive (i.e. L half)

05 if k == i  // found ith element since A[q] is sorted pivot

06  return A[q]

07 if i < k  // ith element is less than pivot and in L

08  return Random-Select(A, p, q-1, i)

09 else  // ith element is greater than pivot and in R

10  return Random-Select(A, q+1, r, i-k)  // important: adjust i when searching right




Select(A, i)
01 use median of medians to partition
02 return ith smallest value in A

Inversion Count

Input: unsorted array A[1…n]
Output: the number of inversions where A[i] > A[j] and i < j
Structure: modify Merge-Sort, anything from R before L is empty in an inversion of (q-p+1) - (i-1)
Run-time: O(n lg n) same as Merge Sort

Pseudocode for Inversion Count



Inversion-Count(A) where A is unsorted array

01 inversionCount = 0  // global variable

02 Merge-Sort(A, 1, n)  // call Merge Sort on entire array

03 return inversionCount




Merge-Sort(A, p, r)
01 if p < r  // standard algo with call to Merge()
...
...
05  Merge(A, p, q, r)


Merge(A, p, q, r])  // modified to count inversions
01 n1 = q - p + 1  // p to q inclusive of q
02 n2 = r - q  // q to r exclusive of q
03 L[1...n1+1], L[1...n2+1] are new arrays  // need extra memory allocation for sentinels
04 for i=1 to n1  // fill L
05  L[i] = A[p+i-1]
06 for j=1 to n2  // fill R
07  R[j] = A[q+j]
08 L[n1+1] = ∞  // add sentinel
09 R[n2+1] = ∞  // add sentinel
10 i=1, j=1 // reset i & j
11 for k=p to r  // loop and count inversions before sorting
12  if L[i] > R[j] and L[i] != ∞  // inversion found
13      inversionCount = inversionCount + (q-p+1) - (i-1)  // update inversion count
12  if L[i] ≤ R[j]  // continue with standard sorting
13      A[k] = L[i]
14      i = i+1
15  else
16      A[k] = R[j]
17      j = j+1

Triple Count

Input: an unsorted array A[1…n] consisting of distinct integers
Output: the count of triples in array where 3 numbers sum to zero
Idea: sort array, then take every pair of numbers and run binary search to find 3rd value
Run-time: O(n2 lg n) // Binary-Search (lg n) called on double loop (n2)

Pseudocode for Triple Count



Triple-Count(A[1…n]) where A is unsorted array

01 Merge-Sort(A)  // need to sort before we can use Binary-Search

02 tripleCount = 0

03 for i=1 to n-1  // loop through all pairs from left to right

04  for j=2 to n

05      x = A[i]

06      y = A[j]

07      z = -(x+y)  // x + y + z = 0, therefore z = -(x+y)

08      if Binary-Search(A,z) != -1  // Binary-Search found a match, therefore triple exists

09          tripleCount = tripleCount + 1

10 return tripleCount

k From Median

Input: an unsorted array A[1…n] and positive integer k ≤ n
Output: k numbers closest in value to the median of
Idea:
* find median
* create new array D to store the absolute distance from median for each element in A (with same index)
* copy A to A’ and call Select(A’, k) to find kth value.
* need to use A’ because Select will semi-sort the array
* Check values in D: if ≤ kth, then return original value stored in A

Run-time:

Pseudocode for k From Median



K-From-Median(A, k) where A is unsorted array

01 medianIndex = ⌊n⁄2⌋

02 medianValue = Select(A, medianIndex)

03 D is new array

04 for i=1 to n

05  D[i] = |A[i] - medianValue|  // distance from median value

06 copy D to K

07 kValue = Select(K, kValue)

08 for i=1 to n

09  if D[i] ≤ kValue

10      print A[i]

Array Contains Integers That Sum to X

Input: an unsorted array A[1…n] of distinct integers and target sum integer x
Output: True/False if array contains two integers that sum to x
Idea:
* x = y + z
* x = y + A[i]
* we are given x and know A[i], therefore need to search for y = x - A[i]
* sort, then use Binary-Search

Run-time: O(n lg n) // Binary-Search (lg n) called on all elements (n)

Pseudocode for Array Contains Integers That Sum to X



Has-Integer-Sum(A[1…n], x) where A is unsorted array

01 Merge-Sort(A)  // important: sort first so we can use BinarySearch

02 for i=1 to n

03  y = x - A[i]

04  if Binary-Search(A, y) != 1  // found a match, T(n) = O(lg n)

05      return true

06 return false

k Largest Less Than Median

Input: an unsorted array A[1…n] of distinct integers and positive integer k < n⁄2
Output: k largest numbers in A that are less than the median
Idea:
* find median
* use select to roughly sort and get small values below median
* use select to roughly sort again and get m-k values below m-k index
* values in m-k to median should be grouped accordingly

Run-time: O(n)

Pseudocode for k Largest Less Than Median



Get-K-Largest-Less-Than-Median(A, k) where A is unsorted array

01 m = ⌊n+1⁄2⌋  // index of median

02 d = Select(A, m)  // dummy value, semi-sorts to get small values below median

03 d = Select(A, m-k-1)  // dummy value, semi-sorts to get m-k values below m-k

04 return A[m-k…m-1]

General Tips for Dynamic Programming

  • try to identify sequence of steps -> focus on last/incremental step
  • use previous steps/calculations to arrive at specific point
  • breakdown information:
    1. subproblem: in words
    2. recurrence: equation -> ALWAYS SOLVE BASE CASES
  • assume you are ending or doing something at A[k]
  • consider transforming data and applying known algorithm solution
  • for contiguous subsequence problems examine subproblems that end at i/j

Knapsack Problem

Input:
* w1,…,wn weight of each item
* v1,…,vn value of each item
* W max weight limit

Output: max possible value that fits into backpack within constraint W
Idea: ith object has 2 possibilities:
1. object exceeds weight limit -> use max value of objects before ith object considered
2. object fits under weight limit:
* more value with object -> add ith value to max value of i-1 objects at weight limit of wj-wi
* more value without obect -> use max value of i-1 objects at same weight limit

Subproblem: M[i,j] = max possible value using v[1…i] and weights w[1…i] where ∑W ≤ j
Recurrence:

cell value
M[i, j] = if w<sub>i</sub> &#8804; j, then max { v<sub>i</sub> + M[i-1, j-w[i]], M[i-1, j] }
else: M[i-1, j]

Base Case:
* M[i, 0] = 0 // no items at weight limit 0
* M[0, j] = 0 // no items if 0 items allowed

Run-time: T(n, W) = O(n * W)

Pseudocode for Knapsack Problem



Max-Value-In-Knapsack(w1,…,wn,v1,…,vn, W)

01 create table M[0…n, 0…W]

02 for i=0 to n

03  M[i,0] = 0

04 for j=0 to W

05  M[0,j] = 0

06 for i=1 to n

07  for j=1 to W

08      if w[i] ≤ j

09          M[i,j] = max{ v[i] + M[i-1, j-w[i]], M[i-1, j] }

10      else

11          M[i,j] = M[i-1, j]

12 return M[n,W]

Longest Common Subsequence

Input: two strings X[1…m], Y[1…n]
Output: the length of the longest common subsequence (same order of letters, not necessarily contiguous)
Idea: adding ith letter to compare X[1…i] and Y[1…j] has 2 possibilities:
1. if X[i] matches Y[j] then there is one more longer subsequence compared to X[1…i-1] and Y[1…j-1]
2. otherwise same length subsequence as either X[1…i-1], Y[1…j] or X[1…i], Y[1…j-1]

Subproblem: M[i,j] = length of longest possible subsequence for X[1…i] and Y[1…j]
Recurrence:

cell value
M[i, j] = if X[i] == Y[j], then M[i-1, j-1] + 1
else: max{ M[i-1, j], M[i, j-1] }

Base Case:
* M[i, 0] = 0 // can't make a subsequence with zero letters
* M[0, j] = 0 // can't make a subsequence with zero letters

Run-time: T(m, n) = O(m * n)

Pseudocode for Longest Common Subsequence



Longest-Common-Subsequence(X[1…m], Y[1…n])

01 create table M[0…m, 0…n]

02 for i=0 to m

03  M[i,0] = 0

04 for j=0 to n

05  M[0,j] = 0

06 for i=1 to m

07  for j=1 to n

08      if X[i] == Y[j]

09          M[i,j] = M[i-1, j-1] + 1

10      else

11          M[i,j] = max{ M[i-1, j], M[i, j-1]}

12 return M[m,n]

Longest Common Substring

Input: two strings X[1…m], Y[1…n]
Output: the length of the longest common substring (contiguous letters in same order)
Idea: the longest common substring that ENDS at X[i], Y[j] has 2 possibilities:
1. if X[i] matches Y[j] then the longest common substring that ENDS at X[i], Y[j] is one more letter long than the substring at X[1…i-1], Y[1…j-1]
2. otherwise the longest common substring that __ENDS_ at X[i], Y[j] is zero

Subproblem: L[i,j] = max length of longest substring that ENDS at X[i], Y[j]
Recurrence:

cell value
L[i, j] = if X[i] == Y[j], then L[i-1, j-1] + 1
else: 0

Base Case:
* L[i, 0] = 0 // can't make a substring with zero letters
* L[0, j] = 0 // can't make a substring with zero letters

Run-time: T(m, n) = O(m * n)

Pseudocode for Longest Common Substring



Longest-Common-Substring(X[1…m], Y[1…n])

01 create table L[0…m, 0…n]

02 for i=0 to m

03  L[i,0] = 0

04 for j=0 to n

05  L[0,j] = 0

06 maxLength = 0

07 for i=1 to m

08  for j=1 to n

09      if X[i] == Y[j]

10          L[i,j] = L[i-1, j-1] + 1

11      else

12          L[i,j] = 0

13      maxLength = max{ maxLength, L[i,j] }

14 return maxLength

Rod Cutting

Input: rod of length n, price per length as array P[1…n]
Output: max revenue that can be generated from rod of length n
Idea:
* a rod of length j can be split at i = 0 to j-1 ways
* assume we know the max value at each split for any possible additional splits
* find the max value using P[i] + M[j-1]
* note that i will cross j-1, so all possible permutations are considered

Subproblem: A[j] = max possible revenue that can be generated from a rod of length j
Recurrence: A[j] = max<sub>i=0 to j</sub>{ P[i] + M[j-i] }
Base Case: A[0] = 0
Run-time: T(n) = O(n2)

Pseudocode for Rod Cutting



Max-Revenue(n, P[1…n])

01 create new array M[0…n]

02 M[0] = 0

03 for j=1 to n

04  maxRevenue = -∞

05  for i=0 to j

06      maxRevenue = max{ maxRevenue, P[i] + M[j-i] }

07  M[j] = maxRevenue

08 return M[n]

Max Interval Sum

Input: array A[1…n] of real numbers
Output: a contiguous subsequence that has maximum sum of all possible contiguous intervals
Idea: Assume we know the max possible interval sum that ENDS at A[i-1]. The max sum that ENDS at A[i] is the max value of 2 possibilities:
1. adding A[i] to the max interval ENDING at A[i-1] or
2. starting a new subsequence at A[i]

Subproblem: M[i] = max interval value that ENDS at A[i]
Recurrence: M[i] = max{ A[i], A[i] + M[i-1] }
Base Case: M[0] = 0
Run-time: T(n) = O(n)

Pseudocode for Max Inteval Sum



Max-Interval-Sum(A[1…n])

01 create new array M[0…n]

02 M[0] = 0

03 maxSum = -∞

04 leftIndex = -1

05 rightIndex = -1

06 for i=1 to n

07  M[i] = max{ A[i], A[i] + M[i-1] }

08  if M[i] > maxSum

09      maxSum = M[i]

10      rightIndex = i

11      if A[i] == maxSum

            leftIndex = i

12 return A[leftIndex…rightIndex], maxSum

Recurrence:

cell value
B[i, j] = if A[i] > j, then B[i-1, j]
else: 1 if B[i-1,j] == 1 or B[i-1, j-A[i]] == 1
don’t use ith or use ith

Base Case:
* B[i, 0] = 1 // subset exists that makes zero
* B[0, j] = 1

Run-time: T(n) = O(n * V)

Recurrence: f<sub>i</sub>[j] = a<sub>i</sub>[j] + min{ f<sub>1</sub>[j-1], f<sub>2</sub>[j-1] }
Base Case: <sub>i</sub>[1] = a<sub>i</sub>[1] + e<sub>1</sub>

02 f1[1] = a1[1] + e1

03 f2[1] = a2[1] + e2

04 l1[1] = 1 // tracks source as line 1 or line 2 (i.e. parent)

05 l2[1] = 2 // tracks source as line 1 or line 2 (i.e. parent)

16      l2[j] = 2

17 if (f1[n] + x1) < (f2[n] + x2)

18  f = f1[n] + x1

19  l = 1

20 else

21  f = f2[n] + x2

22  l = 2

23 return f  \ l tracks path

Can Make Change

Input: unlimited supply of coin denominations d1,…,dn and amount A
Output: True/False if possible to make change for amount A from coins of denomination d1,…,dn
Idea: There are 3 considerations for using di:
1. di exactly forms A
2. we can form A using coins d1,…,di-1
3. we can form A using a combination of di + remainder of A⁄di i.e. A mod di

Subproblem: B[i,j] contains True/False if we can form amount j using coin denominations d1,…,di
Recurrence:

cell value condition
B[i, j] = 1 if: A mod d[i] == 0
B[i-1, j] == 1
B[i, j mod d[i]] == 1
0 otherwise

Base Case:
* B[i, 0] = 0 // can't make change with 0 coins
* B[0, j] = 0

Run-time: T(n) = O(n * A)

01 create table B[0…n, 0…A]

02 for i = 0 to n

03  B[i,0] = 0

04 for j=0 to V

05  B[0,j] = 0

06 for i=1 to n

07  for j=1 to A

08      if (A mod d[i] == 0) or (B[i-1, j] == 1) or (B[i, j mod d[i]] == 1)

09          B[i,j] = 1

10      else

11          B[i,j] = 0

12 return B[n,A]

cell value
